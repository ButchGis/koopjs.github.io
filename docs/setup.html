Setup

Setting up a development environment for Koop

All you need to have installed before Koop will run is a Node.js environment and an internet connection. We recommend using version 0.10 until the Node.js Foundation stabilizes differences between 0.12 and iojs.

Required

  • Node.js

Recommended

  • Postgres and PostGIS

Koop uses an in-memory cache by default. This is not recommended when running in production. A PostGIS cache using koop-pgcache is recommended for all production deployments. See caches for more details.

Using Docker

There is an example of how to get Koop running with PostGIS inside a docker container here.
